# UI Components

High‑level overview of core UI primitives. Stories now export a direct `const meta = { ... }` object (factory removed for CSF index reliability).

## Conventions

- Each component lives under `UI/ComponentName`.
- Prefer minimal props in the primary story; showcase variants in dedicated stories.
- Use lucide icon classes via `<span className="iconify lucide--icon-name" />`.
- Accessibility: ensure interactive elements have discernible labels.

## Components Covered

- Icon
- IconButton
- Button
- Tooltip
- TableEmptyState
- ThemeToggle
- PageTitleHero (now in `Molecules/`) – related to layout titles

## Adding a New UI Component
1. Create the component under `src/components/ui/`.
2. Add `Component.stories.tsx` exporting:
```ts
const meta: Meta<typeof Component> = {
	title: 'UI/Component',
	component: Component,
	parameters: { docs: { description: { component: 'Short description.' } } },
	tags: ['autodocs']
};
export default meta;
```
3. Provide a succinct description (1–2 sentences) about purpose & usage.
4. Add variant stories when they teach something (size, state, color, etc.).

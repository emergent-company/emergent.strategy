# Chat Components

Chat-specific building blocks (composer, action bar, CTA cards) for conversational UX.

## Architecture

- Composer (`ChatPromptComposer`) handles multiline text, privacy mode, submission.
- Actions (`ChatPromptActions`) expose file attach, privacy toggle, and future extensible buttons.
- CTA Cards (`ChatCtaCard`) guide users on empty/new conversation surfaces.

## Usage Guidelines

- Keep composer placeholder actionable: describe user goal (e.g., “Let us know what you need…”).
- Reserve CTA cards for first-time or zero-history states.
- Prefer semantic buttons with aria-labels; icons alone must have accessible labels.

## Adding a Chat Component
Export a literal meta object in `ChatYourComponent.stories.tsx`:
```ts
const meta: Meta<typeof ChatYourComponent> = {
	title: 'Chat/YourComponent',
	component: ChatYourComponent,
	parameters: { docs: { description: { component: 'Short description.' } } },
	tags: ['autodocs']
};
export default meta;
```

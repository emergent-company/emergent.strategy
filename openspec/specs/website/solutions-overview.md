### emergent.product: The Operating System for Product Strategy

**The Domain Problem:**

Product development is inherently complex. You're navigating uncertainty with incomplete information, coordinating distributed teams, and trying to learn fast enough to avoid wasting months on the wrong thing.

**Four failure modes:**

1. **Strategic Drift:** OKRs disconnected from execution. Engineers don't understand "why." Product leaders can't trace outcomes to assumptions.

2. **Siloed Execution:** Strategy docs in one tool, roadmaps in another, PRDs scattered across Notion, evidence buried in analytics dashboards.

3. **Documentation Debt:** Board decks take 10-15 hours to assemble. PRDs are outdated before they're finished. New PMs spend 2-3 weeks trying to understand context.

#### 2. Strategic Agents (Intelligent Agency)

Three AI agents aligned to the READY-FIRE-AIM operating loop:

**Pathfinder Agent (READY - Sense & Frame):**

- Synthesizes opportunity maps from market research, user interviews, competitive analysis
- Proposes OKRs aligned to business goals
- Identifies RATs (critical unknowns blocking each OKR)
- Scaffolds value models and generates dependent Work Packages
- **Impact:** 3 days vs. 3 weeks for solo founder strategy development

**Product Architect Agent (FIRE - Build & Deliver):**

- Guides detailed component modeling (UVPs, metrics, dependencies)
- Maintains traceability mappings (ensures Work Packages link to RATs)
- Validates against EPF schemas (flags inconsistencies)
- Suggests optimizations ("Work Package A blocks Work Package Bâ€”consider reordering")

**Synthesizer Agent (AIM - Measure & Recalibrate):**

- Autonomously ingests data (analytics, user feedback, status updates, A/B tests)
- Compares Actual vs. Planned KRs (are we hitting targets?)
- Updates RAT status (validated? refuted? inconclusive?)
- Generates assessment reports (narrative summary of quarter)
- Proposes calibration memos (what should we adjust for next cycle?)
- **Impact:** 15 minutes vs. 10-15 hours for board deck prep

---

#### 3. The READY-FIRE-AIM Loop (Adaptive Loops)

EPF v2.0.2 operating loop for continuous learning:

**READY (Sense & Frame):**

- Define OKRs, identify RATs, scaffold value models, generate Work Packages
- **Gate:** Strategic framework validated by stakeholders

**FIRE (Build & Deliver):**

- Execute Work Packages, run experiments, gather evidence
- **Continuous learning:** RAT status updates as evidence accumulates
- **Gate:** Quarter ends OR high-confidence evidence triggers early pivot

**AIM (Measure & Recalibrate):**

- Assess KR progress, synthesize learnings, generate calibration memos
- **Decision:** Stakeholders review evidence, approve adjustments
- **Loop back:** Calibrated strategy becomes input to next READY phase

**What it replaces:** Linear roadmaps that become obsolete, quarterly planning cycles that arrive too late

**What it enables:** Evidence-based evolution, systematic learning, early pivots before sunk costs mount

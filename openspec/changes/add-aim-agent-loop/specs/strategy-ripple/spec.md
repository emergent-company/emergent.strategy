## ADDED Requirements

### Requirement: Cycle Trigger Signal Type

The system SHALL support a `cycle_trigger` signal type in the ripple signals
table, used to surface AIM cycle readiness conditions (time elapsed, signal
accumulation, KR staleness) as first-class signals alongside structural
coherence signals.

`cycle_trigger` signals differ from structural signals in two ways:
- They are generated by the AIM trigger evaluation path (not `commit_batch`)
- They are ephemeral: re-evaluated on each page load, not persisted between
  sessions unless the trigger condition remains true

`cycle_trigger` signals have `severity = "info"` by default, escalating to
`"warning"` if the condition has been true for more than 7 days.

#### Scenario: Cycle trigger signal displayed in signal feed

- **WHEN** trigger evaluation fires for an instance
- **AND** the AIM landing page signal feed renders
- **THEN** a `cycle_trigger` signal appears at the top of the feed with the
  trigger reason and a link to the draft action

#### Scenario: Cycle trigger signal cleared

- **WHEN** a new `assessment_report` is committed for the instance
- **THEN** any active `cycle_trigger` signals for that instance are resolved
  automatically as part of the commit_batch post-commit hook
